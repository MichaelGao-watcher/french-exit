# French Exit — 工作电脑安全撤离工具

> **French Exit**（法式撤离）：源自俚语，指不告而别、悄然离场。这里它代表一种体面、干净、无后顾之忧的离职方式——**在交还工作电脑前，把属于你的个人文件带走，把不该留的清理干净。**

---

## 一、GitHub 路径大小写评估

本项目仓库名称建议统一使用全小写 **`french-exit`**。在 GitHub 及 Git 生态中，**路径大小写是有影响的**，建议如下：

| 场景 | 大小写是否敏感 | 影响说明 |
|------|---------------|---------|
| GitHub Web 访问 (`github.com/user/repo`) | **不敏感**（会自动重定向到正确大小写） | 输入 `french-exit` 也能访问，但 URL 会跳转为仓库创建时的实际大小写 |
| Git 版本控制（Linux 服务端 / GitHub Actions） | **敏感** | 如果本地大小写与远程不一致，可能出现文件追踪异常 |
| Windows 本地文件系统 | **默认不敏感** | `French-exit` 与 `french-exit` 在本地被视为同一目录，容易在跨平台协作时引入大小写冲突 |
| macOS APFS | **默认不敏感**（可配置敏感） | 与 Windows 类似，存在潜在冲突 |

### 结论与建议

- **结论**：GitHub 对仓库名在 URL 层面做了兼容重定向，但 Git 底层在 Linux 服务器上是**大小写敏感**的；Windows/macOS 开发者本地默认**大小写不敏感**。混用大小写可能导致克隆异常、CI 脚本失效、子模块路径错误等问题。
- **建议**：**仓库名称与本地文件夹统一使用全小写 `french-exit`**，与 `requirements.txt`、CI 配置中的引用完全一致，彻底杜绝跨平台大小写隐患。

## 二、功能简介

French Exit 是一款面向职场人的**简易桌面工具**，用于在离职或换机前对个人电脑进行安全审查：

1. **输入入职日期** —— 以此为分界线，找出你入职后产生或修改的所有文件。
2. **自由选择扫描范围** —— 自动列出所有可用磁盘（C:\、D:\、E:\），用户勾选要扫的盘符，也可额外添加自定义路径。
3. **设置排除目录与豁免路径** —— 排除目录（如 `node_modules`）在扫描时跳过；豁免路径则完全不受处理，在最终报告中也会明确列出。
4. **自动风险评估** —— 根据文件路径、扩展名、文件名关键词，初步标记 🔴 高 / 🟡 中 / 🟢 低风险。
5. **用户自主决策** —— 对每一个（或每一批）文件，自行选择：
   - **📦 带走** —— 复制到指定输出目录，或**打包成 ZIP**
   - **🗑️ 删除** —— 执行前**自动压缩备份**到临时目录，再移入回收站
   - **⏭️ 忽略** —— 不做任何处理
6. **最终确认界面** —— 所有操作执行完毕后，清晰展示：
   > **"French Exit 已完成。除豁免路径外，所有本地文件均已完成剔除或转移。"**

### 风险分级规则（简要）

| 级别 | 典型特征 | 示例 |
|------|---------|------|
| 🔴 高风险 | 个人照片/视频、聊天记录、敏感证件、简历、银行/工资相关 | `Pictures/`、`微信/`、`resume.pdf`、`身份证.jpg` |
| 🟡 中风险 | 个人文档、下载内容、代码文件、疑似个人项目 | `Downloads/`、`.py` 文件、`github.com/xxx/my-side-project` |
| 🟢 低风险 | 缓存、日志、临时文件、空文件、系统可重下安装包 | `Temp/`、`Cache/`、`.exe` 安装包、`*.log` |

> ⚠️ **注意**：自动评估仅为**初步参考**，最终决策权完全在你。请仔细核对文件路径后再执行删除或带走。

## 三、安装与运行

### 环境要求

- Python 3.8 或更高版本
- Windows / macOS / Linux
- Chrome、Edge 或 Firefox 浏览器（Eel 会优先尝试调用，不可用时回退到系统默认浏览器）

### 第一步：克隆项目

```bash
git clone https://github.com/yourname/french-exit.git
cd french-exit
```

### 第二步：安装依赖

```bash
pip install -r requirements.txt
```

如果系统提示 `pip` 不存在，请先安装 Python（ https://www.python.org/downloads/ ），安装时**勾选 "Add Python to PATH"**。

### 第三步：运行

```bash
python main.py
```

首次运行时会自动进行环境检查：
- ✅ Python 版本是否满足
- ✅ `eel`、`send2trash` 是否已安装
- ✅ 浏览器是否可用

如有缺失，终端会输出**一键修复命令**，复制粘贴运行即可。

## 四、使用步骤

### 1. 配置扫描范围

- **入职日期**：输入 `YYYY-MM-DD`，例如 `2022-03-15`
- **扫描盘符**：勾选要扫描的磁盘（自动检测）
- **自定义路径**：手动添加额外目录，如 `D:\Backup`
- **排除目录**：扫描时跳过这些目录名（默认已含 `node_modules`, `.git`, `venv`）
- **豁免路径**：完全不扫描、不处理的目录，最终报告中会明确列出

### 2. 开始扫描

点击「🔍 开始扫描」，后台线程遍历文件，前端实时显示进度。扫描期间可随时点击「停止扫描」。

### 3. 查看结果并决策

扫描完成后进入结果页：
- 表格按「高风险优先 + 修改时间倒序」排列
- 支持按文件名/路径搜索、按风险等级筛选
- 点击表头可排序
- 勾选文件后批量标记「带走」「删除」「忽略」
- 也可点击「仅选高风险」快速聚焦敏感文件

### 4. 配置带走方式

- **带走输出目录**：指定复制或打包的目标位置（如 U 盘路径）
- **带走方式**：
  - 📁 **原样复制**：按 `盘符/原路径` 结构复制
  - 🗜️ **打包成 ZIP**：所有带走文件压缩为单个 ZIP，便于转移

### 5. 执行操作

点击「⚠️ 执行所选操作」后，弹出**二次确认窗口**，汇总：
- 带走数量、方式、输出目录
- 删除数量、备份位置
- 忽略数量
- 豁免路径清单

确认后：
- **带走**：复制或打包到指定目录
- **删除**：先打包备份到临时目录 → 再移入回收站
- **忽略**：无操作

### 6. 查看完成报告

执行完毕后显示最终界面：

> **French Exit 已完成。除豁免路径外，所有本地文件均已完成剔除或转移。**

同时展示：
- 带走/删除的成功与失败数量
- 备份文件位置
- 操作日志位置
- 可一键打开相关目录

## 五、安全提示

- **删除前自动备份**：所有标记删除的文件会先被压缩成 ZIP 备份到临时目录，即使误删也可恢复。
- **强烈建议安装 `send2trash`**：`pip install send2trash`，这样删除会进入回收站，可二次恢复。
- **豁免路径**：如果你确定某些目录完全不需要处理（如公司共享盘），请提前加入豁免路径。
- **带走前确认输出目录空间充足**，尤其当照片/视频较多时。
- 本工具**只操作本地文件**，不会上传任何数据。

## 六、打包成独立可执行文件（无需安装 Python）

如果你想在没有 Python 环境的电脑上运行：

```bash
pip install pyinstaller
pyinstaller -F -w --add-data "ui/web;ui/web" main.py
```

打包完成后，`dist/main.exe` 可直接双击运行。打包时会自动内嵌前端资源。
