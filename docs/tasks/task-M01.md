# 任务拆分：M01 — frontend（前端）

> 职责：渲染 5 个核心页面，管理全局状态，调用 Tauri Commands，响应后端进度事件。UI 风格：Apple Design，跟随系统深色/浅色模式。

---

## 前置条件

- [ ] INF-03（前端工程骨架完成）
- [ ] INF-07（前后端类型同步完成）
- [ ] M02 commands（所有 IPC command 已实现）

## 推荐开发顺序

1. FE-01 ~ FE-06（基础架构：主题、状态、路由、IPC）
2. FE-07 ~ FE-10（输入页 + 扫描页 + 结果页核心功能）
3. FE-11 ~ FE-13（特殊交互：默认勾选、预览、加密弹窗）
4. FE-14 ~ FE-17（确认页 + 报告页 + 设置）
5. FE-18 ~ FE-19（测试）

---

## 子任务清单

### P1 — 必须做

- [ ] **FE-01** 搭建 Tauri 前端项目（React 18 + TypeScript + Tailwind CSS）
  - 配置 `vite` / `webpack`（Tauri 默认模板）
  - 安装依赖：`react`, `react-dom`, `tailwindcss`, `@tauri-apps/api`
  - 验证 `cargo tauri dev` 能正常渲染 React 组件
  - 测试点：开发服务器启动不报错

- [ ] **FE-02** 配置 Apple Design 主题系统
  - 定义 CSS 变量：`--radius-lg`, `--radius-md`, `--font-sans`, `--glass-bg`, `--glass-border`
  - 基础组件样式：按钮（圆角 8px）、卡片（圆角 12px + backdrop-blur）、输入框
  - 测试点：UI 组件视觉检查通过

- [ ] **FE-03** 实现系统深色/浅色模式跟随
  - 监听 `window.matchMedia('(prefers-color-scheme: dark)')`
  - Tauri 主题 API（如可用）同步
  - Tailwind `dark:` 前缀适配所有组件
  - 测试点：切换系统主题，前端实时响应

- [ ] **FE-04** 实现 IPC 封装层
  - 文件：`src/api/commands.ts`
  - 封装所有 Tauri command 调用：`startScan`, `pauseScan`, `resumeScan`, `getScanResults`, `previewItem`, `submitDecisions`, `startExecution`, `getResourceConfig`, `setResourceConfig`
  - 统一错误处理：后端 `FrontendError` 转为前端可展示的 `AppError`
  - 测试点：每个封装函数 TypeScript 类型正确

- [ ] **FE-05** 实现全局状态管理（Zustand 或 React Context）
  - Store 结构：`{ session, scanResults, decisions, progress, settings, ui }`
  - 状态：会话 ID、扫描结果分页缓存、用户决策 Map、当前进度、资源设置
  - 测试点：状态更新后组件正确重渲染

- [ ] **FE-06** 实现页面路由/状态机映射
  - 不用传统 URL 路由，用状态机驱动页面切换
  - `page: 'input' | 'scanning' | 'results' | 'confirm' | 'report'`
  - 页面切换动画：淡入淡出 200ms
  - 测试点：状态切换时页面正确渲染

- [ ] **FE-07** 实现"入职日期输入"页面
  - 日期选择器：`type="date"`，限制最大值为今天
  - 类别勾选：聊天、浏览器、系统、注册表、文件、开发工具、环境变量（全选默认）
  - "开始扫描"按钮，点击后调用 `startScan`
  - 测试点：非法日期（未来日期）禁止提交

- [ ] **FE-08** 实现"扫描进度"页面
  - 整体进度条（0~100%）
  - 当前扫描器名称和描述（如"正在扫描 Chrome 浏览器数据..."）
  - 已发现数量实时更新
  - 暂停/恢复按钮（调用 `pauseScan` / `resumeScan`）
  - 取消按钮
  - 测试点：模拟进度事件，UI 正确更新

- [ ] **FE-09** 实现"结果清单"页面框架
  - 分类 Tab：聊天 / 浏览器 / 系统 / 注册表 / 文件 / 开发工具 / 环境变量
  - 每页分页（pageSize = 50）
  - 虚拟滚动（`react-window` 或 `@tanstack/react-virtual`）
  - 测试点：1000+ 条目不卡顿

- [ ] **FE-10** 实现结果项的勾选/全选/取消功能
  - 每条 TraceItem 左侧复选框
  - 顶部"全选本页"/"取消全选"按钮
  - 底部"删除"/"保留"/"打包"批量操作按钮
  - 决策状态用 Map 存储：`{ [itemId]: Action }`
  - 测试点：全选后批量操作正确更新状态

- [ ] **FE-11** 实现微信条目默认选中
  - 收到扫描结果时，自动为 `suggested_action === 'DeleteOrPack'` 且 `scanner_id === 'scanner-chat'` 的条目设置 `decision = 'Delete'`
  - UI 上复选框默认打勾，用户可手动取消
  - 这是 RULE-03 硬规则
  - 测试点：微信条目初始状态为已勾选

- [ ] **FE-12** 实现环境变量条目默认不选中
  - 收到扫描结果时，`scanner_id === 'scanner-env'` 的条目默认不勾选
  - 这是 RULE-02 硬规则
  - 测试点：环境变量条目初始状态为未勾选

- [ ] **FE-13** 实现"预览"弹窗
  - 点击条目"预览"按钮触发
  - 文本文件：直接展示内容（前 4KB）
  - 图片文件：展示 Base64 图片
  - 不支持：展示"暂不支持预览此类型"
  - 弹窗样式：Apple Design 模态框（毛玻璃背景 + 圆角）
  - 测试点：各类型预览正确渲染

- [ ] **FE-14** 实现"最终确认"页面
  - 汇总三类决策清单：删除（红色）、打包（蓝色）、保留（灰色）
  - 每项显示：名称、路径、大小
  - 支持最后一刻修改（点击可跳转回结果页）
  - "确认执行"按钮，二次弹窗："以下文件将被彻底删除，无法恢复，是否继续？"
  - 这是 RULE-01 和 RULE-09 硬规则
  - 测试点：二次弹窗文案正确

- [ ] **FE-15** 实现"执行/报告"页面
  - 执行阶段：展示当前操作（"正在安全擦除..."/"正在打包..."）
  - 完成阶段：显示 `generate_summary` 返回的文本摘要
  - 自动跳转：报告完成后短暂停留（3 秒），然后提示"浏览器即将打开庆祝页"
  - 测试点：执行进度正确展示

- [ ] **FE-16** 实现加密文件确认弹窗
  - 打包阶段后端推送加密文件事件
  - 前端展示弹窗："⚠️ 该文件已加密或无法直接打开，打包后可能无法查看，确定要带走吗？"
  - 用户选择"确定"或"跳过"
  - 选择回传给后端
  - 测试点：弹窗触发和回传正确

- [ ] **FE-17** 实现资源占用设置入口
  - 设置图标 → 滑块或开关：CPU 限制 10%/30%/50%/无限制
  - 调用 `setResourceConfig`
  - 默认显示"限制中（30%）"
  - 测试点：设置后后端配置正确更新

- [ ] **FE-18** 【测试】组件测试（React Testing Library）
  - 输入页：日期选择、类别勾选、提交按钮
  - 进度页：进度条更新、暂停/恢复按钮状态
  - 使用 `vitest` + `@testing-library/react`

- [ ] **FE-19** 【测试】E2E 测试（Playwright）
  - 完整流程：输入日期 → 开始扫描（mock）→ 勾选条目 → 提交决策 → 执行 → 验证报告页
  - 由于 Tauri 桌面应用的特殊性，E2E 可能以 Web 模式运行（`tauri dev` 的 Web 端）

### P2 — 后续迭代

- [ ] **FE-20** 实现结果列表搜索与过滤
  - 按文件名/路径关键词搜索
  - 按文件大小范围过滤

- [ ] **FE-21** 实现列表排序（按时间、按大小、按名称）

- [ ] **FE-22** 实现结果项的缩略图预览（图片/视频首帧）
  - 列表中直接展示小缩略图，无需点开弹窗

- [ ] **FE-23** 实现交互动画优化
  - 页面切换：滑动 + 淡入
  - 列表项：hover 微动效
  - 按钮：点击涟漪效果

- [ ] **FE-24** 实现首次使用引导（Onboarding）
  - 遮罩层引导用户完成第一步
