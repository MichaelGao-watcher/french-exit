## 2. 推荐开发阶段（Phase）

## 4. 风险项与应对

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
| Windows Job Object API 限制效果不如预期 | CPU 限制失效，用户电脑卡顿 | 中 | 备选方案：用 `SetPriorityClass` 降低进程优先级 + 手动 `Sleep` 降速 |
| 浏览器数据库被锁定（Chrome 运行中） | 扫描失败或结果不全 | 高 | Scanner 捕获 SQLITE_BUSY，标记 risk_note，不中断流程 |
| 微信目录大小计算耗时（10GB+） | 扫描卡顿，用户以为死机 | 高 | 异步计算大小，先返回条目再后台补全 size_bytes；或显示 "计算中..." |
| 注册表推断误报率高 | 用户不信任结果，体验差 | 中 | P1 保持保守策略（只匹配高置信度模式）；P2 引入权重打分降低误报 |
| 大文件安全擦除极慢（1GB 文件 ≈ 数分钟） | 执行阶段用户以为卡死 | 高 | Orchestrator 实时推送进度；Frontend 展示 "正在安全擦除 xxx / 共 yyy" |
| 前端虚拟滚动实现复杂 | 百万级文件结果页卡顿 | 中 | 已设计分页 + 虚拟滚动；如仍卡顿，P2 增加前端缓存和增量加载 |
| Tauri + Windows 特定 API 兼容性 | 部分 Windows 版本不支持 | 低 | 所有 Win32 API 调用加 `cfg(windows)` 和错误降级 |
